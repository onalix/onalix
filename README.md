My name is Ramon N. Rayas. I'm a data scientist.

I do all my professional work in Jupyter notebooks, from data exploration and prototyping to production code.
I use specialized tooling to extract production code from my notebooks. I can't see myself doing
it any other way. I'm a strong advocate of _literate programming_, which, in my opinion, is the best way to write
and maintain code long term. In this programming style, which was invented by computer scientist Donald Knuth in 1984, the main goal is to write a narrative for humans to read, not code for machines. Every program becomes a work of literature where, chapter by chapter, the writer takes the reader on an exploratory journey. In _literate programming_, documentation is no longer an afterthought, but the core idea. All executable code needed to run the program is extracted from the documentation, not the other way around.
